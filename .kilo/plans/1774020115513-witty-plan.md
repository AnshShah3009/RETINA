# Phase 1: Critical Bugs (HIGHEST PRIORITY)
**Goal**: Fix bugs that cause crashes, incorrect results, or compilation issues

## 1.1 cv-hal Critical Bugs

### 1.1.1 Morton Code Encoding Bug
- **File**: `crates/hal/src/gpu_kernels/mod.rs:44`
- **Issue**: Uses `mx` instead of `my` in bit manipulation
- **Code**: `my = (my | (mx << 4)) & 0x030C30C3;`
- **Fix**: Change to `my = (my | (my << 4)) & 0x030C30C3;`
- **Test**: `crates/hal/tests/morton_encoding_tests.rs` (CREATE)

### 1.1.2 Dispatch Function Signature Mismatch
- **File**: `crates/hal/src/gpu.rs:158-164` vs `crates/hal/src/context.rs:45-51`
- **Issue**: Trait expects `name: &str`, implementation uses `shader_source: &str`
- **Fix**: Align implementation with trait definition
- **Test**: `crates/hal/tests/dispatch_tests.rs` (CREATE)

### 1.1.3 Unsafe Pointer Cast Without Runtime Check
- **Files**: 
  - `crates/hal/src/gpu.rs:1316-1318` (pointcloud transform)
  - `crates/hal/src/gpu.rs:1329` (pointcloud bounds)
  - `crates/hal/src/gpu.rs:2412-2414` (unsafe transmute)
- **Issue**: Uses `TypeId` check but unsafe casts could cause UB
- **Fix**: Add proper runtime type verification or use safe alternatives

### 1.1.4 Potential Panic on `.unwrap()`
- **File**: `crates/hal/src/gpu.rs:199`
- **Issue**: `_uniform_buf.as_ref().unwrap()` when uniforms is empty
- **Fix**: Handle empty uniforms case properly

## 1.2 cv-runtime Critical Bugs

### 1.2.1 Pipeline Kernel Execution Stub
- **File**: `crates/runtime/src/pipeline/mod.rs:205`
- **Issue**: Kernel execution is completely stubbed - accepts params but does nothing
- **Code**: `let _: (_, _, _) = (name, params, device_runtime.as_ref());`
- **Fix**: Implement actual kernel dispatch through cv-hal
- **Test**: `crates/runtime/tests/pipeline_execution_tests.rs` (CREATE)

### 1.2.2 Kernel Fusion Generates Placeholder Shaders
- **File**: `crates/runtime/src/pipeline/fusion.rs:31-58`
- **Issue**: `generate_fused_shader` creates non-functional WGSL with comments
- **Fix**: Generate actual executable WGSL code
- **Test**: `crates/runtime/tests/fusion_tests.rs` (CREATE)

### 1.2.3 CPU Fallback Logic Bug
- **File**: `crates/runtime/src/orchestrator.rs:345`
- **Issue**: Fallback triggered but runs on same device instead of switching to CPU
- **Code**: `return f();` without switching device

- **Test**: `crates/runtime/tests/cpu_fallback_tests.rs` (CREATE)

### 5.4.1 ICP Optimization ✅ COMPLETED
- **Issue**: O(n²) brute force correspondence search at 443ms for 100K points
- **Solution**: Implemented spatial hash grid for local neighborhoods
- **Files created**: `crates/hal/src/gpu_kernels/spatial.rs`
- **Performance Results**:
  | Points | Brute Force | Spatial Hash | Speedup |
  |--------|-------------|--------------|---------|
  | 1K | 15ms | 67ms | 0.2x (slower) |
  | 10K | 132ms | 46ms | 2.9x |
  | 100K | 443ms | 53ms | 8.4x |
- **Notes**: Spatial hash is slower for small datasets (hash table overhead) but much faster for large datasets
- **Test**: `spatial_icp_perf` module in `perf_tests.rs`

### 5.4.2 Lanczos4 Optimization ✅ COMPLETED
- **Issue**: 102ms for 1024x1024 was above threshold
- **Solution**: Simplified shader (removed redundant calculations, precomputed offsets)
- **File modified**: `crates/hal/shaders/lanczos4.wgsl`
- **Performance Results**:
  | Resolution | Before | After | Improvement |
  |-----------|--------|-------|-------------|
  | 512x512 → 256x256 | 4.6ms | 4.7ms | same |
  | 1024x1024 → 512x512 | 102ms | 29ms | **3.5x faster** |
